# A股 Scoring Framework

所有 A股 报告共用的固定评分公式，禁止主观拍分。  
可用 `scoring.py` 脚本验算全套分数（见 SKILL.md 脚本快速参考）。

---

## 1 必须输出的评分区块

每份报告必须包含以下四个区块：

1. **评分输入摘要** — 列出所有计算所需的原始输入
2. **影响评分（0–100）** — 五个维度 + 综合环境分
3. **置信度分解** — 四个维度 + 置信度总分 + 等级
4. **评分结果摘要** — 一行综合判断

使用代理指标时，在对应行标注 `[代理]`。

---

## 2 必填输入字段

| 输入 | 说明 |
| ---- | ---- |
| 价格相对 20MA / 50MA / 200MA | 各自是否在上方（bool） |
| RSI(14) | 数值 |
| MACD 柱体 | 正 / 负 |
| 价格相对 Pivot | 是否在 Pivot 上方（bool） |
| 可用项 / 应有项 | 整数，用于数据完整性 |
| 双源一致项 / 可校验项 | 整数，用于多源一致性 |
| 关键新闻条数 | 整数 |
| 是否存在高冲击事件 | bool |
| 关键缺口项数 | 整数，用于惩罚 |
| 明显分歧项数 | 整数，用于惩罚 |

---

## 3 固定公式

### 3.1 数据完整性

```
数据完整性 = round(100 × 可用项 / 应有项)
```

### 3.2 多源一致性

```
多源一致性 = round(100 × 双源一致项 / 可校验项)
```

### 3.3 技术结构强度

| 子项 | 计分规则 | 满分 |
| ---- | -------- | ---- |
| MA 对齐 | 价在 20MA 上方 +10，50MA 上方 +10，200MA 上方 +10 | 30 |
| RSI 区间 | RSI < 40 → 8；40–60 → 15；> 60 → 25 | 25 |
| MACD 动量 | 柱体 > 0 → 25；否则 → 10 | 25 |
| Pivot 位置 | 价 ≥ Pivot → 20；否则 → 8 | 20 |

```
技术结构强度 = MA对齐 + RSI区间 + MACD动量 + Pivot位置   (max 100)
```

### 3.4 风险偏好温度

```python
if 广度数据可得:
    base = clamp(广度分, 0, 60)          # 广度口径
else:
    base = clamp(20 + (ETF日涨跌% / 3) × 20, 0, 40)  # 代理口径，标注[代理]

风险偏好温度 = clamp(base + 新闻风险修正, 0, 100)
# 新闻风险修正范围：-20 到 +20
```

### 3.5 其他三项基础维度

以下三项各以 50 分为基础：

```
宏观与流动性支持 = clamp(50 + 10×正向证据数 - 10×负向证据数, 0, 100)
商品与通胀扰动   = clamp(50 + 10×正向证据数 - 10×负向证据数, 0, 100)
事件冲击可控度   = clamp(50 + 10×正向证据数 - 10×负向证据数, 0, 100)
```

---

## 4 缺口与分歧惩罚

```
缺口惩罚   = min(25, 5 × 关键缺口项数)
分歧惩罚   = min(20, 5 × 明显分歧项数)

各维度最终分 = clamp(原始分 - 缺口惩罚 - 分歧惩罚, 0, 100)
```

缺口惩罚和分歧惩罚**对每个维度独立施加**。

---

## 5 综合环境分

```
综合环境分 = 0.30 × 技术结构强度
           + 0.20 × 风险偏好温度
           + 0.20 × 宏观与流动性支持
           + 0.15 × 商品与通胀扰动
           + 0.15 × 事件冲击可控度
```

---

## 6 置信度总分

```
技术信号清晰度 = round(0.5 × 技术结构强度 + 0.5 × 风险偏好温度)
事件可解释性   = 事件冲击可控度

置信度总分 = 0.35 × 数据完整性
           + 0.25 × 多源一致性
           + 0.25 × 技术信号清晰度
           + 0.15 × 事件可解释性
```

---

## 7 等级映射

| 分数 | 等级 | 含义 |
| ---- | ---- | ---- |
| ≥ 75 | High | 数据充分，结论可信 |
| 55–74 | Medium | 基本可用，保守解读 |
| < 55 | Low | 数据严重缺失，仅供参考 |

---

## 8 输出要求

- 每个分数必须配一句读法（不能只给分数）。
- 综合环境分与置信度总分必须同时出现。
- 若分数为 Low，必须在结论中标注"请以实盘行情终端数据为准"。

---

## 9 一眼可执行映射（必填）

为保证简报"一眼能用"，每份报告在评分后必须追加一个 4 行区块：

1. 市场状态：{{偏多 / 中性 / 偏弱}}
2. 执行节奏：{{进攻 / 均衡 / 防守 / 观望}}
3. 关键位与触发：{{1 个关键位 + 上破/下破条件}}
4. 风险提示：{{当日最大不确定性}}

执行节奏由综合环境分与置信度等级联合决定：

| 综合环境分 | 置信度 | 执行节奏 |
| ---------- | ------ | -------- |
| ≥ 70 | High / Medium | 进攻 |
| 55–69 | High / Medium | 均衡 |
| 40–54 | 任意 | 防守 |
| < 40 | 任意 | 观望 |
| 任意 | Low | 观望（覆盖上表） |

说明：该映射仅用于市场环境解读，不构成买卖建议。
