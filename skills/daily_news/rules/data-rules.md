# A股 Data Rules

所有 A股 报告共用的底层规则，任何报告类型都必须执行。

---

## 1 数字获取顺序

每一个数字按以下顺序处理，不得跳步：

1. 先做实时网页搜索。
2. 双源交叉校验。两源一致直接使用；轻微差异用区间展示；明显分歧同时展示两值并注明"来源有分歧，结论按保守口径"。
3. 实时不可得（或时间戳超过 30 分钟）时，退回前收数据，搜索格式：`[标的] 前收 [日期]`。
4. 仍不可得时明确写缺失，给出官方来源入口链接，字段不得留空。

**绝对禁止：先写后查 / 编造 / 猜测 / 外推 / 使用评论文章的二手数字。**

---

## 2 来源优先级

**行情数据：**

1. 上交所 / 深交所 / 中金所 官方页面
2. 东方财富 (eastmoney.com)
3. 同花顺 (10jqka.com.cn)
4. 新浪财经 (finance.sina.com.cn)
5. 腾讯自选股 / 腾讯财经 / 雪球
6. Investing / Yahoo Finance / Google Finance（仅全球资产补充）

**新闻来源（同时覆盖）：**

财联社、华尔街见闻、第一财经、界面新闻、中国证券报、证券时报、上海证券报、上交所公告、深交所公告、东方财富快讯、同花顺资讯、Reuters、Bloomberg

---

## 3 新鲜度标签

所有数字必须带标签：

| 延迟 | 标签 |
| ---- | ---- |
| ≤ 30 分钟 | `[实时]` |
| 30 分钟 – 3 小时 | `[延迟 HH:MM CST]` |
| > 3 小时 | 不用作实时，改用 `[前收: DD MMM YYYY]` |

格式示例：`3,245 [实时] Source: 东方财富 (eastmoney.com)`

---

## 4 新闻规则

每条新闻仅写四行，格式固定：

```
发生了什么：一行。
市场影响：一行。
[阅读全文 →](direct article URL)
发布时间：[HH:MM CST DD MMM YYYY]  Source: [publication]
```

- 不得给首页链接或搜索页链接。
- 直链不可得时写：`链接不可用 — 请在 [source] 搜索 [headline]`

---

## 5 代理指标规则

主指标不可得时可用代理，但必须在数字旁标注 `[代理]`：

| 主指标 | 代理指标 |
| ------ | -------- |
| A50 期货 | 上证50现货 或 可验证 A50 CFD |
| 沪金 / 沪银主力 | 伦敦金银 或 COMEX 金银 |
| 期权 PCR / Max Pain | 300ETF 价格 + 成交变化 + 波动率 |

---

## 6 缺口与分歧集中披露

同一模块缺失 ≥ 2 项时：
- 模块内只保留可得关键数字与结论。
- 所有缺失项统一移到报告末尾的"数据可得性与缺口"区块，不在每个字段重复写"暂缺"。

---

## 7 可读性规则

1. 先输出"3 行核心判断"（市场状态 / 关键位与触发条件 / 执行重点）。
2. 每条信息包含"数字或结论 + 含义"，避免裸数字。
3. 每个模块保留 5–8 条核心信息，不堆字段。
4. 有分歧时不展开数据细节，只给保守口径结论。
5. 先给"一屏摘要"再展开：摘要不超过 8 行。
6. 单条要点最多两句，优先短句和表格，不写大段背景。
7. 重点新闻默认 3–5 条；仅在用户要求详细时扩展。

---

## 8 A股内容优先级

内容稀缺时，按此顺序取舍：

1. 赚钱效应与市场温度（涨跌家数、涨停跌停、成交额、高位股反馈）
2. 风格强弱（上证 = 权重、创业板 = 成长、中证1000 = 题材情绪、沪深300 = 机构资产）
3. 主线板块与龙头反馈
4. 资金面（北向、成交额、主力资金）
5. 关键指数与关键位（三锚：上证 / 创业板 / 沪深300）
6. 外部扰动（USD/CNY、原油、金银）

技术指标在以上内容都完成后才填。

---

## 9 结论后必须附加的区块

每份报告结论后必须出现：

**行动框架**（三行固定格式）：
- 优先观察什么
- 何时说明环境改善
- 何时说明环境转弱

**触发条件表**（至少 3 行）：

| 触发场景 | 盘面确认信号 | 环境解读 | 观察动作 |
| -------- | ------------ | -------- | -------- |

**数据可得性与缺口**：集中列出本次所有缺失项。

---

## 10 最低可用输出

即使数据严重受限，也必须完成简报。最低要求：

- 核心指数（至少 3 个）
- 至少 1 个外部扰动指标
- 至少 3 条新闻
- 关键位判断
- 数据可得性与缺口区块
