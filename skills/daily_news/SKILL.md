# A股 Report Skill

## 目录结构

| 文件 | 职责 |
| ---- | ---- |
| `rules/data-rules.md` | 数据获取规则、来源优先级、新鲜度标签、缺口处理、可读性规则 |
| `rules/scoring.md` | 影响评分与置信度分解的固定公式 |
| `prompts/intraday.md` | A档 — 日内交易报告规格 |
| `prompts/swing.md` | B档 — 波段交易报告规格 |
| `prompts/full.md` | C档 — 全量市场框架报告规格 |
| `scripts/scoring.py` | 脚本：从命令行输入计算全套评分 |
| `scripts/pivot.py` | 脚本：从 OHLC 计算经典 Pivot 支撑阻力位 |
| `reports/` | 生成的简报存档目录，按日期+类型命名 |

---

## Step 1 — 确认报告类型

如果用户**没有**明确说要 A / B / C，第一条回复只输出以下文字，然后等待回复：

```
你好。你今天是哪种交易者？

  A) 日内交易   — 赚钱效应、风格强弱、主线板块、关键位、今日新闻。
  B) 波段交易   — 周趋势、板块轮动、北向资金、融资融券、周度新闻。
  C) 全量报告   — 日内 + 周度 + 衍生品 + 资金 + 事件完整全景。

请回复 A、B 或 C。
```

如果用户已经指定类型，跳过提问，直接进入 Step 2。

---

## Step 2 — 加载规则

1. 读取 `rules/data-rules.md` — **所有类型**都要应用。
2. 读取 `rules/scoring.md` — **所有类型**都要应用。
3. 按用户类型读取对应规格文件：
   - A → 读取 `prompts/intraday.md`
   - B → 读取 `prompts/swing.md`
   - C → 读取 `prompts/full.md`

---

## Step 3 — 执行报告

按 Step 2 加载的规则生成报告。  
报告模板位于 `templates/`：

| 类型 | 模板 |
| ---- | ---- |
| A 高可读版 | `templates/A股晨会简报_A_高可读模板.md` |
| B 波段版 | `templates/A股晨会简报_B_波段模板.md` |
| C 全量版 | `templates/A股晨会简报_C_全量模板.md` |

---

## Step 4 — 保存简报

报告生成完毕后，将完整内容保存到 `reports/` 目录。

**文件命名规则：**

```
reports/YYYY-MM-DD_<类型>.md
```

| 类型 | 示例文件名 |
| ---- | ---------- |
| A 日内交易 | `reports/2026-04-10_A_日内.md` |
| B 波段交易 | `reports/2026-04-10_B_波段.md` |
| C 全量报告 | `reports/2026-04-10_C_全量.md` |

**保存规则：**
- 日期取报告生成当日（CST），格式 `YYYY-MM-DD`。
- 若当日同类型已存在文件，追加后缀 `_2`、`_3` 以此类推，不覆盖。
- 文件第一行写入元数据注释：`<!-- type: A|B|C  date: YYYY-MM-DD  generated: HH:MM CST -->`
- `reports/` 目录不存在时自动创建。

## 内容优先级（所有类型统一）

1. 赚钱效应与市场温度
2. 风格强弱
3. 主线板块与龙头反馈
4. 资金面
5. 关键指数与关键位
6. 外部扰动

---

## 通用约束

- 先结论，后展开。
- 不编造数据，不留无来源数字。
- 缺失项集中披露，不在每个字段重复写"暂缺"。
- 只做市场环境解读，不给买卖建议。

---

## 简洁可读模式（默认启用）

- 默认输出"先简后全"：先给**一屏可读摘要**，再给结构化明细。
- 一屏可读摘要固定 5 行：
  1) 市场状态（偏多/中性/偏弱）
  2) 今日/本周主导风格（权重/成长/题材）
  3) 执行节奏（进攻/均衡/防守/观望）
  4) 最关键位与触发条件（1 行）
  5) 当下执行重点（1 行）
- 各模块优先"结论 + 1 个关键数字 + 含义"，禁止长段背景叙述。
- 新闻默认精简：仅保留最重要的 3–5 条（全量报告最多 8 条）。
- 缺失数据统一放在"数据可得性与缺口"，正文不重复刷屏。
- 若用户明确要求"展开/详细版"，再补充完整扩展内容。

## 脚本快速参考

计算 Pivot（输入昨日 H/L/C）：

```bash
python .github/skills/a-share/scripts/pivot.py 4622.83 4509.12 4563.54
```

计算完整评分：

```bash
python .github/skills/a-share/scripts/scoring.py \
  --rsi 48.7 --above-pivot \
  --available 14 --total 20 \
  --consistent 10 --verifiable 12 \
  --gaps 2 --divergences 1 \
  --macro-bull 1 --commodity-bear 1
```
